## Plan: Art Tool Copilot Agent Execution Plan

目标是在4周内交付 Windows 首发的桌面端图像处理 MVP，支持批量 PNG 透明边缘裁剪，并预留后续功能扩展接口。

### 项目边界
- 包含：批量导入本地文件、透明边缘裁剪、输出到指定目录、任务队列、错误汇总、扩展接口。
- 包含（扩展预留）：为图像格式转换、图像压缩、图像修复提供统一接入契约、参数校验规范和功能路由。
- 不包含（当前版本）：真实格式转换算法、真实图像压缩算法、真实图像修复算法、云端能力、账号系统、多平台发布。

### 技术决策
- 框架：Tauri + Rust + 前端框架。
- 首发平台：Windows。
- 输出规则：保持原文件名输出到指定目录，重名自动追加序号。
- 界面复杂度：专业版（功能区+参数区+队列区+结果区）。

### 执行流程（线性）
1. 初始化工程基础：搭建 Tauri 项目、前后端通信与统一命令入口。
2. 定义可扩展处理器契约：完成 Processor、Registry、Context、Result、Params，以及 validate() 和 process() 约定。
3. 实现透明边缘裁剪能力：完成 alpha 扫描、BBox 计算、裁剪保存，并覆盖全透明图和无透明边缘图等边界场景。
4. 构建输入输出管线：实现 PNG 文件扫描过滤、输出目录管理与重名追加序号策略。
5. 实现批处理调度器：完成受控并发、任务状态机、进度回传与单文件失败隔离。
6. 打通专业版 UI 闭环：完成功能选择、参数设置、输入输出配置、任务队列监控与结果汇总，并预留格式转换/压缩/修复的参数面板入口。
7. 建立扩展骨架：注册 FormatConvertProcessor、CompressProcessor、RepairProcessor 的空壳实现，仅启用参数校验与路由，不启用实际算法。
8. 完成质量保障与优化：补齐单元/集成/性能测试，调优并发默认值与内存占用，并验证三类扩展占位能力可正常编排。
9. 完成交付与文档：完成 Windows 打包和使用文档，补齐扩展开发文档（如何新增格式转换、压缩、修复类处理器）。

### 核心任务清单（Agent 可直接执行）
1. 创建核心模块目录：core、commands、tests、docs。
2. 定义 Processor 接口与参数校验契约。
3. 实现透明边缘裁剪算法与结果保存流程。
4. 实现批处理调度器与任务状态机。
5. 打通前后端任务启动、进度监听、结果展示。
6. 实现报告导出：成功数、失败数、耗时、错误原因。
7. 设计并实现三类扩展占位处理器：格式转换、图像压缩、图像修复（仅接口与校验）。
8. 为三类扩展补齐参数模型与错误码规范，确保后续可无缝接入真实算法。
9. 完成测试基线与回归用例（含扩展占位路由测试）。
10. 完成打包与验收清单。

### 验收标准
- 支持批量处理至少100张 PNG，流程稳定。
- 失败文件可追踪且不影响其他文件完成。
- 输出目录结果正确，重名策略生效。
- UI 可完成从导入到导出的完整闭环。
- 图像格式转换、图像压缩、图像修复三类扩展入口可在系统中被识别并完成参数校验与任务路由。
- 在干净 Windows 环境安装后可正常运行。

### 风险与缓解
- 大图内存占用高：限制并发并做分批调度。
- 损坏图片导致异常：输入预检+单任务隔离。
- 路径兼容问题：统一路径规范并覆盖中文路径测试。
